## 关于 `PacDocs`

`PacDocs` 是使用 [`Nuxt`](https://github.com/nuxt/nuxt) 构建的，网站风格也是缝缝补补，基本借鉴了 `Nuxt` 官网，但是其中有不少 `PacDocs` 自己的理念和组件。

PacDocs 使用了以下一些开源项目:

:icon{name="i-ph-number-one-duotone"} 核心:

- :Icon{name="i-simple-icons-nuxtdotjs"} [`Nuxt`](https://github.com/nuxt/nuxt) - 直观的 Vue.js 框架
- :Icon{name="i-simple-icons-nuxtdotjs"} [`Nuxt/UI`](https://github.com/nuxt/ui) - `Nuxt` 官方团队维护的 UI 组件库
- :Icon{name="i-simple-icons-nuxtdotjs"} [`Nuxt/Content`](https://github.com/nuxt/content) - 适用于 `Nuxt` 的基于文件的 CMS
- :Icon{name="i-simple-icons-nuxtdotjs"} [`Nuxt/Studio`](https://nuxt.studio/) - 与团队一起在生产环境中编辑 Nuxt 网站
- :Icon{name="i-simple-icons-tailwindcss"} [`tailwindcss`](https://tailwindcss.com/) - 只需书写 HTML 代码，无需书写 CSS，即可快速构建美观的网站
- :Icon{name="i-logos-vueuse"} [`VueUse`](https://vueuse.org/) - Vue.js 组合式 api 工具集合
- :Icon{name="i-simple-icons-nginx"} [`Nginx`](https://nginx.org/) - 高性能的开源 Web 服务器
- :Icon{name="i-simple-icons-nodedotjs"} [`Node.js`](https://nodejs.org/en) - 基于Chrome V8引擎的 JavaScript 运行时环境

:icon{name="i-ph-number-two-duotone"} 组件:

- [`Vidtor`](https://github.com/Vanessa219/vditor) - 浏览器端的 Markdown 编辑器，支持所见即所得（富文本）、即时渲染（类似 Typora）和分屏预览模式
- [`ChatGPT-Next-Web`](https://github.com/pacmandoh/ChatGPT-Next-Web) - 拥有你自己的跨平台 ChatGPT 应用
- [`ChatGPT-Next-Web-LangChain`](https://github.com/Hk-Gosuto/ChatGPT-Next-Web-LangChain) - 让 `ChatGPT-Next-Web` 支持 `GPT4.0-VISION(视觉)`，以及基于 [LangChain](https://www.langchain.com/) 实现的插件功能(搜索、计算、网络请求、Wiki、DALL-E 3、StableDiffusion、Arxiv...)
- [`Channel.io`](https://developers.channel.io/docs) - 在线客户支持和实时消息传递平台

::callout{icon="i-ph-lightbulb-duotone"}
PacDocs 离不开这些开源项目的支持，如有条件请前往项目网站或 [GitHub](https://github.com/) 赞助这些组织和作者一杯咖啡 ☕️
::

## 结构和功能

  - **进阶指南** - 基于**指南**中的内容的进阶部分，更深层次的理解
  - **示例** - PacDocs 中的示例在此处进行汇总，相当于 PlayGround 游乐场
  - **笔记** - 不成体系的碎片化知识和功能实现

- **集成**
  - **模块** - 基于 `Python` 或 `R` 构建的在线分析工具 (正在开发中...)

- **资源**
  - **介绍** - 为**资源**板块提供一份板块地图，让您快速了解其中包含什么
  - **软件推荐** - 一些魔法 🧙‍♂️ 使用的软件推荐，以及如何安装，搭配 `PacAssets` 可直接点击链接下载，或者点击按钮: :show-assets 去下载站查找下载
  - **随笔心得** - 记录一些心得看法、感悟
  - **GitHub库** - 优秀的 GitHub 开源库推荐

- **博客** - `PacDocs` 最新的更新内容、特性介绍，以及 `Nuxt` 和 `Vue` 等的更新推文总结

## 一些问题

PacDocs 本质上是基于内容 (Content) 构建的网页应用，使用的是 `Nuxt/Content` 提供的 CMS (Content Management System, 内容管理系统)，所以目前为止没有考虑使用后端，如后期需求量增加会考虑使用 :Icon{name="i-simple-icons-python"} `Python` Web 框架 - :Icon{name="i-simple-icons-flask"} `Flask` 作为 PacDocs 的后端。

本人前端理解还比较浅薄，如果存在什么问题或者建议请[邮件我](mailto:pacmandoh@outlook.com)。或者点击 :showChannel 直接给我讯息
